## Interpretation

Q4 loads and the gateway stays healthy, but 130k-context operation is too slow
for Hermes-agent routing in the current environment. The Q4 timeout is a
practical usability failure, not a load failure.

Q2 is materially better than Q4 for warm requests and completes the
Hermes-routing preset, but it remains much slower than the Qwen3.5 2B baseline.
Observed VRAM after Q2 load/benchmark was about `5.4 GiB / 6 GiB`, so Q2's
smaller GGUF file does not remove the large 130k-context runtime footprint.

The next meaningful retest should run with the machine otherwise idle and with
one variable changed at a time:

- lower context size, for example `32768` or `65536`
- newer llama.cpp image or a source build
- QAT profile without other Docker workloads
- direct llama.cpp server request without Hermes/browser containers

Until then, keep `qwen3.5-2b-q4-xl` or `gemma4-e2b-q4` as practical local-agent
defaults.
