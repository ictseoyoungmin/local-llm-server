## Interpretation

The model loads and the gateway stays healthy, but 130k-context operation is
too slow for Hermes-agent routing in the current environment. The timeout is a
practical usability failure, not a load failure.

The next meaningful retest should run with the machine otherwise idle and with
one variable changed at a time:

- lower context size, for example `32768` or `65536`
- newer llama.cpp image or a source build
- QAT profile without other Docker workloads
- direct llama.cpp server request without Hermes/browser containers

Until then, keep `qwen3.5-2b-q4-xl` or `gemma4-e2b-q4` as practical local-agent
defaults.
